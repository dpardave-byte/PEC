# Skill: PEC Git y Entrega

Antes de cerrar:

1. Revisar git status.
2. Revisar git diff cuando sea util.
3. Identificar archivos modificados.
4. Separar cambios propios de cambios previos del usuario.
5. No hacer commit sin autorizacion.
6. No usar git add . si hay archivos no relacionados.
7. Sugerir mensaje de commit claro.

Formato de entrega:

Resumen de trabajo realizado

1. Archivos revisados:
2. Archivos modificados:
3. Cambios principales:
4. Pruebas realizadas o sugeridas:
5. Riesgos o advertencias:
6. Proximo paso recomendado:
7. Mensaje de commit sugerido:
